# The Index

## לקוח-שרת (Client / Server)

> מודל ארכיטקטורת תקשורת שבו צד אחד (הלקוח) שולח בקשות, וצד שני (השרת) מקבל אותן, מעבד ומחזיר תשובה.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)

* **אפשרות א' (סועד ומטבח במסעדה):** אתם יושבים בשולחן (הלקוח), בוחרים מנה מהתפריט ונותנים הזמנה למלצר. המטבח (השרת) מכין את המנה ומחזיר אותה. אתם לא צריכים לדעת איך הבישול עובד – רק לבקש ולקבל.
* **אפשרות ב' (ATM וסניף הבנק):** המכשיר ברחוב (הלקוח) לא מחזיק את הכסף שלכם. הוא שולח בקשה לשרתי הבנק (השרת) בזמן אמת, מקבל אישור ומשחרר מזומן. כל הלוגיקה והנתונים – יושבים על השרת.

### 💻 קוד תמציתי (Python)

```python
import requests

# CLIENT side: sending a request to a server
response = requests.get("https://api.livecode.org/users/42")

# SERVER returns a response object with status and data
if response.status_code == 200:         # 200 = success
    user = response.json()
    print(f"Hello, {user['name']}!")    # Hello, Gal!
else:
    print(f"Error: {response.status_code}")
```

> **לשים לב:** הלקוח תמיד מתחיל את השיחה – השרת מחכה ומגיב. זה ההבדל הבסיסי: הלקוח *דורש*, השרת *נותן שירות*. הבנת הכיוון הזה עוזרת מאוד כשמדברים על אבטחה ותכנון מערכות.
