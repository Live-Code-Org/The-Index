## פרוטוקולים (Protocols)

> שפה מוסכמת מראש בין שני צדדים, שמגדירה כיצד להעביר מידע ביניהם – מה סדר הדברים, באיזה פורמט וכיצד מטפלים בטעויות.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)

* **אפשרות א' (חוקי תנועה):** כל הנהגים בעולם הסכימו שאדום = עצור, ירוק = לנוע. אף אחד לא ייצר את ההסכמה הזו לבד – היא עובדת כי *כולם* פועלים לפיה. פרוטוקול הוא בדיוק אותה הסכמה, רק בין מחשבים.
* **אפשרות ב' (פרוטוקול דיפלומטי):** כשמנהיגי מדינות נפגשים, יש נוהל ברור: מי מושיט יד ראשון, מי מדבר ראשון, באיזו שפה. בלי הנוהל הזה – כאוס. זה בדיוק מה שהפרוטוקול עושה לתקשורת בין מחשבים.

### 💻 קוד תמציתי (Python)

```python
import json

# Two computers agree on a shared data format (JSON protocol)
user_data = {
    "name": "Gal",
    "role": "student",
    "active": True
}

serialized = json.dumps(user_data)   # Converting to agreed format for transmission
received    = json.loads(serialized)  # Decoding back from the agreed format

print(received["name"])  # Gal
```

> **לשים לב:** פרוטוקולים הם לב האינטרנט. כשאתם כותבים `https://` בדפדפן, אתם בוחרים פרוטוקול ספציפי לתקשורת. כל "http", "ftp", "smtp" – כולם פרוטוקולים שונים, כל אחד מתאים לסוג תקשורת שונה.

### 🛠️ יישום בפרויקט

בבניית API: כל שרת שכותבים מגדיר לעצמו "פרוטוקול" – בפורמט JSON, עם שדות מסוימים, בסדר מסוים. הלקוח שמתחבר חייב לפעול לפי אותן הגדרות. זה בדיוק מה שעושה ה-Documentation של API.
