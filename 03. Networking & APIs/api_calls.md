# קריאות API (API Calls)

כדי שהאפליקציה שלנו תוכל להשתמש בשירותים קיימים בעולם (כמו נתוני מזג אוויר, סליקת אשראי או שליחת SMS), היא לא צריכה לפתח אותם מאפס. היא פשוט צריכה לדעת "לדבר" עם האפליקציות האחרות שמציעות את השירותים האלו.

---

## ⚙️ קריאות API (API Calls)
>חוזה תקשורת המאפשר לאפליקציה אחת לבקש שירות, לבצע פעולה או לקבל מידע מאפליקציה אחרת (שרת), מבלי להכיר את קוד המקור שלה מבפנים.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (המלצר במסעדה):** אתם הלקוח (הקוד שלכם), המטבח הוא השרת החיצוני, והתפריט הוא התיעוד (Documentation). אתם לא נכנסים לבשל במטבח; אתם בוחרים מנה מהתפריט, מוסרים למלצר את ההזמנה (Request), והוא חוזר מהמטבח עם הצלחת או עם הודעה שהמנה נגמרה (Response).
* **אפשרות ב' (שקע החשמל בקיר):** כדי להפעיל טלוויזיה, אתם לא צריכים להבין איך עובדת תחנת הכוח של חברת החשמל. אתם פשוט מכניסים את התקע לשקע בקיר (ה-API) ומקבלים את החשמל (השירות).

### 💻 קוד תמציתי (Python)
```python
import requests

# שליחת בקשת GET לקבלת מידע משרת חיצוני
response = requests.get("https://api.service.com/v1/data")
data = response.json()  # המרת התשובה למילון פיטוני נוח לעבודה
print(data["result"])
```

#### כלים ומושגים נוספים:
* **מתודות (Methods):** סוג הפעולה שאנחנו מבקשים לעשות. למשל: `GET` לקבלת מידע, ו-`POST` לשליחה או יצירה של מידע חדש.
* **מפתח API (API Key):** הסיסמה או הקוד האישי שלנו מול השרת החיצוני שמשמש לזיהוי ולאבטחת הבקשה.
* **כותרות (Headers):** ה"מעטפה" של הבקשה שבה אנו שמים מידע טכני (כמו מפתח ה-API או סוג המידע).

**דוגמה לשליחת מידע חדש (POST):**
```python
import requests

url = "https://api.shop.com/v1/orders"
headers = {"Authorization": "Bearer YOUR_API_KEY"} # מפתח הזיהוי שלנו
payload = {"item": "Piano", "price": 5000}       # המידע שאנו רוצים ליצור

res = requests.post(url, headers=headers, json=payload)
receipt = res.json() # קבלת אישור ההזמנה שחזר מהשרת
```

### 🛠️ יישום בפרויקט
בבניית אפליקציית חדשות או מזג אוויר: במקום לעדכן את הכתבות והטמפרטורה ידנית בקוד, האפליקציה תבצע קריאת API פשוטה בכל פעם שהמשתמש פותח אותה, ותציג על המסך את הנתונים העדכניים ביותר שחזרו מהשרת.
