## קריאות API

>חוזה תקשורת המאפשר לאפליקציה אחת לבקש שירות או מידע מאפליקציה אחרת (שרת), בלי לדעת איך היא עובדת מבפנים.



### אנלוגיה מהחיים
<div class="rtl">

**המלצר.**  אתה הלקוח (הקוד), המטבח הוא השרת, והתפריט הוא  התיעוד (Documentation)

אתה לא נכנס לבשל; אתה בוחר מנה מהתפריט, נותן למלצר הזמנה (Request), והוא חוזר מהמטבח עם צלחת או עם הודעה שהמנה נגמרה (Response).
<div>

### קוד תמציתי (JS)

```js
const res = await fetch('https://api.service.com/v1/data'); // שליחת בקשה לנתיב ספציפי בשרת
const data = await res.json(); // המרת התשובה הגולמית למבנה נתונים קריא (JSON)
console.log(data.result); // שימוש בנתון הספציפי שחזר מה-API
```


> **לשים לב:** אנו נגדיר משתנים באופן שונה בהתאם למטרת המשתנה:

- נגדיר  באמצעות `const`: עבור ערכים קבועים שלא נרצה שיוכלו לעבור עריכה חופשית. (כמו שם, כתובת )
- נגדיר באמצעות  `let`:  עבור ערכים שחייבים להשתנות באופן תדיר (כמו ערכים בלולאות, או ערך שמתעדכן).
