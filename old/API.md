# The Index


## קריאות API

>חוזה תקשורת המאפשר לאפליקציה אחת לבקש שירות או מידע מאפליקציה אחרת (שרת), בלי לדעת איך היא עובדת מבפנים.



### אנלוגיה מהחיים
<div class="rtl">

**המלצר.**  אתה הלקוח (הקוד), המטבח הוא השרת, והתפריט הוא  התיעוד (Documentation)

אתה לא נכנס לבשל; אתה בוחר מנה מהתפריט, נותן למלצר הזמנה (Request), והוא חוזר מהמטבח עם צלחת או עם הודעה שהמנה נגמרה (Response).
</div>

### קוד תמציתי (JS)

```js
const res = await fetch('https://api.service.com/v1/data'); // שליחת בקשה לנתיב ספציפי בשרת
const data = await res.json(); // המרת התשובה הגולמית למבנה נתונים קריא (JSON)
console.log(data.result); // שימוש בנתון הספציפי שחזר מה-API
```

<div class="rtl">

### כלים נוספים
- **מפתח API (API Key):** הקוד האישי שלך מול השרת. הוא משמש לזיהוי, אבטחה ותשלום.
- **מתודות (Methods):** ה"פעולה" שאנחנו מבצעים. `GET` כדי לקבל מידע, `POST` כדי לשלוח/ליצור מידע חדש.
Headers: ה"מעטפה" של הבקשה. שם אנחנו שמים את מפתח ה-API ואומרים לשרת איזה סוג מידע אנחנו שולחים.

</div>


**לדוגמה -**

<div dir="">

```js


const res = await fetch('https://api.shop.com/v1/orders', { // שליחת מידע חדש (POST) עם מפתח אבטחה
  method: 'POST', // אנחנו יוצרים משהו חדש, לא רק מבקשים
  headers: { 
    'Authorization': 'Bearer YOUR_API_KEY', // המפתח שמאשר שזה אנחנו
    'Content-Type': 'application/json' // אומרים לשרת: "הנה מידע בפורמט JSON"
  },
  body: JSON.stringify({ item: "Piano", price: 5000 }) // המידע עצמו שנשלח
});

const receipt = await res.json(); // התשובה שחוזרת מהשרת (למשל מספר הזמנה)
```

</div>

## יישום בפרויקט

בבניית **אפליקציית חדשות:** אנחנו לא רוצים שהמסך יהיה לבן וקפוא בזמן שהאפליקציה מורידה את הכתבות מהאינטרנט. אנחנו מציגים למשתמש את מבנה האתר (סידור החדר) ורק כשהכתבות חוזרות מהשרת (הבקבוק התמלא) – אנחנו "מזריקים" אותן למסך.
