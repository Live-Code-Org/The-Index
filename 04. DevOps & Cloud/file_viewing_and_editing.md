# צפייה ועריכת קבצים (Viewing & Editing Files)

אחד היתרונות הגדולים של עבודה בשרתים או בטרמינל הוא היכולת לבדוק מה כתוב בתוך קובץ או לערוך אותו בשניות, מבלי להמתין שתוכנות כבדות (כמו VS Code או עורכים גרפיים אחרים) ייפתחו.

---

## ⚙️ cat (Concatenate)
>הדפסת כל תוכן הקובץ ישירות על גבי מסך הטרמינל.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (הצצה למכתב):** קיבלתם מכתב במעטפה. אתם לא רוצים לשמור אותו בארכיון או לערוך אותו, רק לפתוח, לקרוא מהר את הדף היחיד שיש בפנים, ולהמשיך הלאה.
* **אפשרות ב' (צילום רנטגן מהיר):** כמו להעביר מוצר תחת סורק ברקוד שמראה לכם מיד את הפרטים שלו על המסך בלי לפתוח אריזות מורכבות.

### 💻 קוד תמציתי (Bash)
```bash
cat server.js  # הצגת כל קוד המקור של הקובץ server.js ישירות בטרמינל
```

### 🛠️ יישום בפרויקט
נשתמש ב-`cat` כדי לקרוא במהירות קבצי הגדרות קטנים (כמו `.env` או `package.json`) ולוודא שהערכים שכתובים בהם נכונים.

## ⚙️ nano
>עורך טקסט פשוט וקל המופעל ישירות בתוך חלון הטרמינל.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (פנקס כיס קטן):** אתם הולכים ברחוב וצריכים לרשום משהו מהר. במקום לפתוח מחברת ציור ענקית (VS Code), אתם שולפים פנקס קטן מהכיס האחורי של המכנסיים, רושמים את מה שצריך, סוגרים ומחזירים לכיס.
* **אפשרות ב' (הלוח המחיק של המקרר):** מקום פשוט לכתוב בו תזכורות או רשימת קניות ישירות במטבח, בלי לפתוח מחשב או אפליקציה מורכבת.

### 💻 קוד תמציתי (Bash)
```bash
nano config.env # פתיחת הקובץ config.env לעריכה (ליציאה ושמירה: Ctrl+X ואז Y)
```

### 🛠️ יישום בפרויקט
כשמתחברים לשרת מרוחק (כמו AWS או Heroku) וצריכים לשנות שורה בודדת בקובץ ההגדרות, `nano` הוא הכלי המהיר ביותר לעשות זאת מבלי להוריד את הקובץ למחשב האישי שלנו.

## ⚙️ less
>צפייה בתוכן קבצים ארוכים בצורה נוחה המאפשרת גלילה (למעלה/למטה) וחיפוש, מבלי להציף את מסך הטרמינל.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (קריאת ספר אלקטרוני):** במקום לשפוך את כל דפי הספר על הרצפה בבת אחת, אתם מקבלים מסך קורא ספרים (Kindle) שמציג לכם עמוד אחד בכל פעם ומאפשר לכם לדפדף בקצב שלכם.
* **אפשרות ב' (חלון גלילה מבוקר):** כמו להסתכל דרך עינית קטנה על מגילה ארוכה – אתם מזיזים את המגילה למעלה או למטה כדי לראות חלקים שונים ממנה.

### 💻 קוד תמציתי (Bash)
```bash
less big_database.sql # צפייה בקובץ ענק (לניווט: חצים; לחיפוש: / ; ליציאה: Q)
```

### 🛠️ יישום בפרויקט
כשצריכים לעבור על קבצי קוד ענקיים או פלטים ארוכים של פקודות מערכת, שימוש ב-`less` מונע מהטרמינל שלנו להיחנק ולהתמלא באלפי שורות מיותרות.

## ⚙️ head
>הצגת השורות הראשונות בלבד של קובץ טקסט (כברירת מחדל: 10 השורות הראשונות).

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (תקציר בגב הספר):** אתם רוצים לדעת על מה הספר אבל אין לכם זמן לקרוא אותו. אתם קוראים רק את הפסקה הראשונה כדי להבין את הכיוון הכללי.
* **אפשרות ב' (כותרות העיתון):** במקום לקרוא את כל הכתבות בעיתון, אתם מסתכלים רק על השורות הראשיות בעמוד הראשון כדי לקבל את תמצית החדשות.

### 💻 קוד תמציתי (Bash)
```bash
head logs.txt    # הצגת 10 השורות הראשונות של הקובץ
head -n 5 app.js # הצגת 5 השורות הראשונות של הקובץ בלבד
```

### 🛠️ יישום בפרויקט
כאשר קובץ מכיל כותרות עמודות או הגדרות שנמצאות תמיד בראש הקובץ (כמו ייבוא ספריות בפייתון או ג'אווה-סקריפט), נשתמש ב-`head` כדי לראות אותן במהירות.

## ⚙️ tail
>הצגת השורות האחרונות בלבד של קובץ טקסט (כמו לוגים שמתעדכנים בזמן אמת).

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (שורת המבזקים בטלוויזיה):** לא מעניין אתכם מה קרה אתמול; אתם רוצים לראות את העדכונים האחרונים ביותר שקופצים ממש עכשיו בתחתית המסך.
* **אפשרות ב' (עקבות בחול):** אתם הולכים בעקבות מישהו. אתם לא צריכים לראות איפה הוא התחיל לצעוד לפני קילומטר, אתם מסתכלים רק על הצעדים האחרונים שהוא השאיר ממש לפני רגע.

### 💻 קוד תמציתי (Bash)
```bash
tail server.log    # הצגת 10 השורות האחרונות בקובץ
tail -f server.log # מעקב חי (f = follow) אחרי שורות חדשות שמתווספות ללוג ברגע זה!
```

### 🛠️ יישום בפרויקט
בזמן פיתוח שרתים, נפתח טרמינל ייעודי עם `tail -f` על קובץ הלוגים של השרת. בכל פעם שמשתמש יבצע פעולה באתר – נראה את השורה החדשה קופצת מיד על המסך בזמן אמת, מה שמקל מאוד על מציאת באגים.
