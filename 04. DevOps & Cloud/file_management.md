# ניהול קבצים ותיקיות (File & Directory Management)

במקום להשתמש בעכבר כדי ללחוץ על "חדש ⬅️ תיקייה" או "קליק ימני ⬅️ העתק", אנחנו יכולים לנהל את כל הקבצים והתיקיות של הפרויקט שלנו ישירות מהמקלדת. זה מהיר יותר, יעיל יותר, ומאפשר לנו לעבוד על שרתים מרוחקים שאין להם ממשק גרפי בכלל.

---

## ⚙️ touch
>יצירת קובץ חדש וריק בתיקייה הנוכחית (או עדכון תאריך השינוי האחרון של קובץ קיים).

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (דף חלק במחברת):** דמיינו שאתם פותחים מחברת ושמים אצבע על דף ריק חדש. עדיין לא כתבתם עליו כלום, אבל הדף עצמו כבר קיים שם ומחכה לעט שלכם.
* **אפשרות ב' (הטבעת חותמת תאריך):** כמו פקיד בדואר שמניח חותמת "נתקבל" על מעטפה. אם הקובץ כבר קיים, הפקודה `touch` לא תמחוק אותו אלא פשוט תעדכן את חותמת הזמן שלו לרגע הנוכחי.

### 💻 קוד תמציתי (Bash)
```bash
touch index.js      # יצירת קובץ ריק חדש בשם index.js במיקום הנוכחי
touch config.json   # יצירת קובץ הגדרות ריק
```

### 🛠️ יישום בפרויקט
בתחילת פיתוח של פיצ'ר או רכיב חדש, נשתמש ב-`touch` כדי ליצור במהירות את קבצי הקוד הנדרשים (כמו `component.js` או `styles.css`) לפני שנתחיל לכתוב בהם.

## ⚙️ mkdir (Make Directory)
>יצירת תיקייה חדשה בנתיב המבוקש.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (הנחת קלסר ריק על המדף):** אתם רוצים לארגן את מסמכי הארנונה שלכם. אתם הולכים לחנות, קונים קלסר פלסטיק ריק, שמים עליו מדבקה "ארנונה 2026" ומניחים אותו על המדף. הקלסר ריק כרגע, אך הוא מוכן לקבל מסמכים.
* **אפשרות ב' (יצירת מגירה חדשה בארון):** דמיינו נגר שמייצר מגירה חדשה בארון הבגדים שלכם כדי שתוכלו להתחיל למיין בה גרביים.

### 💻 קוד תמציתי (Bash)
```bash
mkdir src         # יצירת תיקייה חדשה בשם src במיקום הנוכחי
mkdir -p src/css  # יצירת תיקיית css ותיקיית src העוטפת אותה בבת אחת
```

### 🛠️ יישום בפרויקט
בניית מבנה פרויקט נקי דורשת חלוקה לתיקיות (כמו `components`, `utils`, `assets`). נשתמש ב-`mkdir` כדי לבנות את שלד התיקיות הזה בקלות.

## ⚙️ cp (Copy)
>העתקה של קובץ או תיקייה למיקום אחר במערכת.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (מכונת צילום):** אתם לוקחים דף נייר עם מתכון לעוגה, מניחים אותו במכונת צילום ולוחצים "העתק". עכשיו יש לכם שני דפים זהים לחלוטין – אחד נשאר אצלכם ואחד אתם מביאים לחבר.
* **אפשרות ב' (שיבוט):** כמו פעולת "העתק-הדבק" (Ctrl+C ואז Ctrl+V) שכולנו מכירים, המשכפלת את המקור למקום היעד.

### 💻 קוד תמציתי (Bash)
```bash
cp info.txt info_backup.txt  # העתקת קובץ info.txt לקובץ גיבוי חדש
cp -r src src_backup         # העתקת תיקייה שלמה (r = recursive) על כל תכולתה
```

### 🛠️ יישום בפרויקט
לפני שמבצעים שינוי גדול או מסוכן בקוד, מומלץ להשתמש ב-`cp` כדי ליצור עותק גיבוי מהיר של קובץ ההגדרות, כדי שאם משהו ישתבש – נוכל לחזור למצב הקודם ברגע.

## ⚙️ mv (Move / Rename)
>העברה של קובץ או תיקייה למיקום אחר, או שינוי השם שלהם.

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (שינוי תווית על קופסה או הזזתה):** אם אתם מזיזים קופסה מהסלון למחסן – זו העברה. אם אתם משאירים אותה באותו מקום אבל מוחקים את הכיתוב "נעליים" וכותבים במקום זה "צעצועים" – זה שינוי שם (Rename). בלינוקס, שתי הפעולות האלו הן אותה פעולה בדיוק.
* **אפשרות ב' (גזירה והדבקה):** כמו פעולת "גזור והדבק" (Ctrl+X ואז Ctrl+V) שמעבירה את הקובץ המקורי למיקום החדש מבלי להשאיר עותק מאחור.

### 💻 קוד תמציתי (Bash)
```bash
mv old_name.js new_name.js  # שינוי שם הקובץ (הקובץ נשאר באותו מקום)
mv app.js src/app.js        # העברת הקובץ app.js לתוך תיקיית src
```

### 🛠️ יישום בפרויקט
נשתמש ב-`mv` כאשר אנחנו מארגנים מחדש את מבנה הפרויקט (למשל, מעבירים קבצי עיצוב לתוך תיקיית `styles`) או משנים שמות של קבצים כדי שיתאימו למוסכמות הפרויקט.

## ⚙️ rm (Remove)
>מחיקה קבועה של קבצים או תיקיות מהמחשב (ללא מעבר בסל המחזור!).

### 🎭 אפשרויות לאנלוגיה (לבחירה ידנית)
* **אפשרות א' (מגרסת נייר משרדית):** דף שנכנס למגרסת נייר מושמד מיד ולא ניתן לשחזור. אין פה "סל מחזור" שאפשר לפתוח ולהוציא ממנו את הקובץ בחזרה. נמחק – נעלם.
* **אפשרות ב' (משאית זבל ישירה):** במקום לזרוק לפח הביתי (סל המחזור), אתם זורקים את הדבר ישר לתוך מכבש משאית הזבל.

### 💻 קוד תמציתי (Bash)
```bash
rm temp.log      # מחיקת קובץ בודד
rm -rf node_modules  # מחיקה כוחנית של תיקייה שלמה על כל תכולתה (זהירות!)
```

### 🛠️ יישום בפרויקט
במהלך הפיתוח נצברים קבצי זבל, לוגים ישנים או תיקיות תלויות פגומות (כמו `node_modules`). נשתמש ב-`rm` כדי לנקות אותם ולפנות מקום.
